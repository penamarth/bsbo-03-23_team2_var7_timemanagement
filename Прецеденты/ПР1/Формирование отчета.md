# Формирование отчета

**Краткое описание:** Руководитель инициирует создание отчета на основе выполненных заданий за указанный период. Система фильтрует задания по заданным критериям, создает объект "Отчет" и сохраняет его в базе данных.

**Основной поток:**
1. Руководитель в пользовательском интерфейсе (UI) нажимает кнопку "Сформировать отчет".
2. UI отображает форму для ввода параметров отчета (период, исполнитель, проект).
3. Руководитель заполняет необходимые критерии и нажимает "Сформировать".
4. Система получает запрос с параметрами и вызывает модуль заданий.
5. Модуль заданий обращается к базе данных с запросом на получение заданий со статусом "Выполнено", которые удовлетворяют заданным критериям (период, исполнитель, проект).
6. База данных возвращает модулю заданий отфильтрованный список выполненных заданий.
7. Модуль заданий передает полученный список заданий в модуль отчетности.
8. Модуль отчетности на основе списка заданий вычисляет статистику (например, общее количество задач, общее затраченное время, процент выполнения и т.д.).
9. Модуль отчетности создает новый объект бизнес-сущности "Отчет", включающий в себя введенные параметры, список заданий и рассчитанную статистику.
10. Модуль отчетности сохраняет созданный объект "Отчет" в базе данных.
11. Система через UI уведомляет руководителя о том, что отчет успешно сформирован и сохранен.

**Альтернативные потоки:**
A1.**Нет данных:** На шаге 6 возвращается пустой список заданий. Система уведомляет руководителя через UI: "За выбранный период по заданным критериям выполненные задания не найдены" и предлагает изменить параметры отчета.

**Предусловия:** Руководитель авторизован в системе. В системе существуют выполненные задания.
**Постусловия:** В базе данных создан и сохранен новый объект "Отчет". Руководитель уведомлен о результате операции.
