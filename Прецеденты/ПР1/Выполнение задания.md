# Выполнение задания

**Краткое описание:** Исполнитель выполняет назначенное задание и фиксирует прогресс через систему.

**Основной поток:**
1. Исполнитель в UI открывает раздел "Мои задания".
2. Система (через Модуль заданий) загружает из базы данных и отображает список назначенных заданий.
3. Исполнитель выбирает задание и нажимает "Начать выполнение".
4. Модуль заданий обновляет в базе данных статус задания на "В работе".
5. Исполнитель регулярно через UI вносит затраченное время и комментарии о прогрессе.
6. Модуль заданий фиксирует эти данные в базе данных.
7. После завершения работ исполнитель нажимает "Завершить задание".
8. Модуль заданий изменяет статус задания на "Выполнено" в базе данных.
9. Модуль уведомлений информирует руководителя о завершении задания.

**Альтернативные потоки:**
- A1. **Запрос помощи:** На шаге 5-6 исполнитель использует UI для создания запроса в модуль заданий, который фиксирует проблему и уведомляет руководителя.

- A2. **Нарушение сроков:** Модуль заданий при периодической проверке обнаруживает просроченное задание и автоматически уведомляет руководителя.

**Предусловия:** Исполнитель авторизован. Задание назначено на него.
**Постусловия:** Статус задания = "Выполнено". Время и прогресс зафиксированы. Руководитель уведомлен.
