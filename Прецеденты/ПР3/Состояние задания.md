# Состояние задания

**Краткое описание**: Авторизованый пользователь видит состояние, историю задания, а так же может вносить изменения

### Основной поток:
1. Пользователь выбирает задание из списка доступных заданий
2. Система отображает детальную информацию о выбранном задании
3. Система показывает текущее состояние задания из возможных вариантов:
   - "Не начато"
   - "В работе"
   - "Выполнено"
   - "Просрочено"
4. Система отображает историю изменений состояния задания


5. Пользователь нажимает "История состояний" на странице задания
6. Система загружает историю изменений состояний задания
7. Система отображает список всех изменений состояний в хронологическом порядке
8. Для каждого изменения отображается:
    - Предыдущее состояние
    - Новое состояние
    - Дата и время изменения
    - Пользователь, изменивший состояние
9. Пользователь может фильтровать историю по дате или пользователю


10. Администратор нажимает кнопку "Изменить состояние"
11. Система отображает доступные для выбора состояния задания
12. Администратор выбирает новое состояние из доступных вариантов
13. Администратор подтверждает изменение
14. Система проверяет корректность перехода между состояниями
15. Система сохраняет новое состояние задания
16. Система записывает изменение в историю состояний
17. Система показывает сообщение об успешном изменении

### Альтернативные потоки:
- **А1:** Задание не найдено
  - Система показывает сообщение: "Задание не найдено"
  - Прецедент завершается

- **А2:** Нет доступа к заданию
  - Система показывает сообщение: "Доступ запрещен"
  - Прецедент завершается

**Предусловия:** Пользователь авторизован в системе  
**Постусловия:** Система отображает текущее состояние задания
