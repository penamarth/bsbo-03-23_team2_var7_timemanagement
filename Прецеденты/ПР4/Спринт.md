# Спринт

**Краткое описание**: Работа с заданиями в рамках текущего спринта.

**Основной поток**:
1. Пользователь выбирает раздел "Спринт"
2. Система отображает спринты в которые включён пользователь
3. Система отображает задачи текущего спринта
4. Пользователь просматривает прогресс спринта
5. Система показывает выполненные и оставшиеся задания
6. Пользователь может фильтровать задания по различным критериям
7. Система обновляет данные в реальном времени

8. Руководитель создаёт спринт
9. Руководитель открывает настройки спринта
10. Система отображает текущие параметры спринта
11. Руководитель изменяет необходимые параметры (даты, цели, участники)
12. Система проверяет корректность изменений
13. Руководитель сохраняет изменения
14. Система применяет новые параметры спринта


15. Пользователь открывает спринт
16. Система загружает все задачи спринта
17. Система группирует задачи по статусам (To Do, In Progress, Done)
18. Пользователь просматривает задачи в каждой группе
19. Система отображает метрики спринта (прогресс, сгорание задач)
20. Пользователь может детализировать любую задачу


21. Руководитель назначает Deadline спринта
22. Руководитель имеет возможность изменять дату окончания спринта
23. Система проверяет корректность изменений
24. Руководитель сохраняет изменения
25. Система применяет новые параметры спринта
26. Система отображает для пользователей Deadline спринта
27. Система фиксирует окончание спринта
28. Руководитель проверяет полноту выполнения спринта
29. Руководитель отмечает невыполненные задачи спринта
30. Руководитель завершает спринт


**Альтернативные потоки**:
- Если спринт не активен, система предлагает создать новый
- При завершении спринта система формирует итоговый отчет
- Если спринт просрочен, система выделяет его цветом

**Предусловия**: Пользователь авторизован, существует активный спринт
**Постусловия**: Пользователь получил информацию о состоянии спринта
