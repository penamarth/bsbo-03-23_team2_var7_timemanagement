# Управление проектами

### Описание
Руководитель вносит изменения в параметры существующего проекта, процесс включает проверку корректности данных и сохранение изменений

### Актёры
- **Руководитель (Director)**: Выбирает проект, инициирует изменения, вносит и подтверждает правки
- **Система (System)**: Предоставляет данные для редактирования, проверяет и сохраняет изменения

### Предусловия
- Руководитель аутентифицирован в системе
- Проект существует и доступен для редактирования
- Руководитель имеет необходимые права для изменения выбранного проекта

### Постусловия
- Изменения в проекте сохранены в системе
- Статус и данные проекта обновлены

### Основной успешный сценарий
1. Руководитель открывает раздел "Проекты"
2. Система отображает список всех проектов, доступных руководителю
3. Руководитель выбирает нужный проект и инициирует действие "Редактировать"
4. Система предоставляет форму с текущими данными проекта
5. Руководитель вносит необходимые изменения (например, в описание)
6. Руководитель сохраняет изменения
7. Система проверяет корректность введенных данных
8. Если данные корректны, система обновляет запись о проектп
9. Если данные некорректны, система возвращает ошибку с описанием обнаруженных проблем
10. Система сохраняет обновленную информацию о проекте
11. Система подтверждает руководителю успешное сохранение изменений и отображает обновленные данные

**Альтернативные потоки:**
- **A1. Проект заблокирован для редактирования**
  Система показывает предупреждение и запрещает редактирвоание
