# Управление заданиями

**Краткое описание:** Руководитель управляет заданиями через пользовательский интерфейс: просматривает, редактирует параметры, отслеживает прогресс выполнения
Пользовательский интерфейс взаимодействует с backend, который выполняет проверку данных, обновляет записи и возвращает результаты

**Основной поток:**
1. Система отображает в пользовательском интерфейсе список всех заданий, доступных руководителю
2. Руководитель выбирает конкретное задание из списка
3. Система(UI) отправляет запрос на сервер с указанием номера выбранного задания и ID руководителя
4. Backend получает запрос и обращается к базе данных для получения детальной информации
5. Backend возвращает в UI данные о задании (описание, сроки, статус, исполнитель, история изменений)
6. Система (UI) отображает полученную информацию о задании на экране
7. Руководитель выбирает нужное ему действие (редактировать параметры, изменить сроки, назначить исполнителя, закрыть задание и т.п.)
8. Система (UI) отображает окно (форму) для выполнения выбранного действия
9. Руководитель вносит необходимые изменения и подтверждает их, нажимая кнопку «Сохранить»
10. Система (UI) отправляет обновленные данные на backend
11. Backend выполняет валидацию полученных данных: проверяет корректность формата, отсутствие конфликтов по срокам и т.д.
12. Если данные корректны, обновляется запись о задании в базе данных и при необходимости изменяется его статус
13. Если данные некорректны, backend возвращает ошибку с описанием обнаруженных проблем (возврат к шагу 9)
14. Backend возвращает в UI подтверждение успешного обновления и обновленные данные задания
15. Система (UI) отображает сообщение об успешном применении изменений и обновлённые данные задания

**Альтернативные потоки:**
- Если задание заблокировано для редактирования, система показывает предупреждение
- При изменении сроков система проверяет конфликты с другими заданиями
- Если задание выполнено, система ограничивает доступные действия
- При ошибке валидации система показывает подробное описание проблемы

**Предусловия:** Руководитель авторизован, задание существует и руководитель имеет к нему доступ
**Постусловия:** Изменения применены к заданию и сохранены в базе данных, при необходимости статус обновлен, руководитель получил визуальное подтверждение операции
