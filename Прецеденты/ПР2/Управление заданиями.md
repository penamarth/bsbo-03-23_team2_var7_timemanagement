# Управление заданиями

### Описание
Руководитель вносит изменения в параметры существующего задания (редактирование, изменение сроков, закрытие). Процесс включает проверку корректности данных и сохранение изменений.

### Актёры
- **Руководитель (Director)**: Выбирает задание, инициирует изменения, вносит и подтверждает правки
- **Система (System)**: Предоставляет данные для редактирования, проверяет и сохраняет изменения

### Предусловия
- Руководитель аутентифицирован в системе
- Задание существует и доступно для редактирования
- Руководитель имеет необходимые права для изменения выбранного задания

### Постусловия
- Изменения в задании сохранены в системе
- Статус и данные задания обновлены

### Основной успешный сценарий
1. Руководитель открывает раздел "Проекты"
2. Система отображает список всех проектов, доступных руководителю
3. Руководитель выбирает нужный проект
4. Система отображает список заданий внутри проекта
5. Руководитель выбирает задание из общего списка
6. Система отображает детальную информацию о задании
7. Руководитель инициирует действие "Редактировать"
8. Система предоставляет форму с текущими данными задания
9. Руководитель вносит необходимые изменения (например, в описание или сроки)
10. Руководитель сохраняет изменения
11. Система проверяет корректность введенных данных
12. Если данные корректны, система обновляет запись о задании
13. Если данные некорректны, система возвращает ошибку с описанием обнаруженных проблем
14. Система сохраняет обновленную информацию о задании
15. Система подтверждает руководителю успешное сохранение изменений и отображает обновленные данные

**Альтернативные потоки:**
- **A1. задание заблокировано для редактирования**
  Система показывает предупреждение
- **B1. Задание выполнено**
  Система ограничивает доступные действия
