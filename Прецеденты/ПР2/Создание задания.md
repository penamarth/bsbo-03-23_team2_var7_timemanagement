# Создание задания

### Описание
Руководитель создаёт новое задание в системе

### Актёры
- **Руководитель (Director)**: Создает задание, заполняет параметры, сохраняет задание
- **Система (System)**: Предоставляет форму для создания, проверяет данные, сохраняет задание

### Предусловия
- Руководитель аутентифицирован в системе
- Руководитель имеет права на создание заданий

### Постусловия
- Создано новое задание
- Все задния сохранены в базе данных с уникальными идентификаторами
- Задание доступно для назначения исполнителей

### Основной успешный сценарий
1. Руководитель открывает раздел "Проекты"
2. Система отображает список всех проектов, доступных руководителю
3. Руководитель выбирает проект из общего списка проектов и в нем инициирует создание задания
4. Руководитель инициирует создание нового задания
5. Система отображает форму для ввода данных задания
6. Руководитель заполняет обязательные параметры: название, описание, срок выполнения
7. Руководитель может назначить исполнителя сейчас, или сделать это позже (в соответсвтии с прецендентом "Выбор исполнителя")
8. Руководитель сохраняет задание
9. Система проверяет корректность введенных данных
10. Система сохраняет задание в базе данных
11. Система подтверждает успешное создание задания
12. Система отображает созданное задание в общем списке заданий проекта

### Альтернативные потоки
- **A. Отмена создания задания**
    1. На шагах 6-8 руководитель отменяет создание задания
    2. Система закрывает форму без сохранения данных
    3. Система возвращает руководителя к общему списку заданий

- **B. Ошибка валидации данных**
    1. На шаге 8 система обнаруживает ошибки в данных (незаполненные обязательные поля, некорректные сроки)
    2. Система выделяет поля с ошибками и показывает сообщения с описанием проблем
    3. Система сохраняет частично заполненные данные в форме
    4. Руководитель исправляет ошибки и повторяет попытку сохранения
