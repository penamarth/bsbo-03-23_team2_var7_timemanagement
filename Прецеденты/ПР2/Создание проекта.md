# Создание проекта

### Описание
Руководитель создаёт новый проект в системе учёта рабочего времени
Проект позволяет группировать задания по теме или клиенту
После создания проекта руководитель может добавлять задания и назначать им исполнителей

### Актёры
- **Руководитель (Director)**: Создает проект, заполняет параметры, сохраняет задание
- **Система (System)**: Предоставляет форму для создания, проверяет данные, сохраняет проект и задания

### Предусловия
- Руководитель аутентифицирован в системе
- Руководитель имеет права на создание проекта

### Постусловия
- Создан новый проект
- Проект сохранены в базе данных с уникальными идентификаторами

### Основной успешный сценарий
1. Руководитель открывает раздел "Проекты"
2. Система отображает список всех проектов, доступных руководителю
3. Руководитель инициирует создание нового проекта
4. Система отображает форму создания проекта
5. Руководитель вводит: название проекта и краткое описание
6. Руководитель нажимает кнопку "Сохранить проект"
7. При успешной проверке система создаёт запись проекта с уникальным ID
8. Система подтверждает успешное создание проекта и добавляет его в общий список проектов

### Альтернативные потоки
- **A. Ошибка валидации данных**
    1. На шаге 5 исистема обнаруживает ошибки в данных (незаполненные обязательные поля, некорректные сроки)
    2. Система выделяет поля с ошибками и показывает сообщения с описанием проблем
    3. Система сохраняет частично заполненные данные в форме
    4. Руководитель исправляет ошибки и повторяет попытку сохранения

- **B. Отмена создания проекта**
    1. На шагах 10-11 руководитель отменяет создание проекта
    2. Система закрывает форму без сохранения данных
    3. Система возвращает руководителя к общему списку проектов
