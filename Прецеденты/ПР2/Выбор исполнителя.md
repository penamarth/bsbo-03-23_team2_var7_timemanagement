# Выбор исполнителя

**Краткое описание:** Руководитель назначает исполнителя на выбранное задание

**Основной поток:**
1. Руководитель инициирует процесс назначения исполнителя для выбранного задания
2. Backend анализирует базу пользователей и формирует список подходящих исполнителей (в соответствии с выбранным ранее заданием)
3. Система(UI) отображает список подходящих кандидатов с краткой информацией (имя, должность, текущая загрузка, активные задания)
4. Руководитель выбирает исполнителя из списка
5. Система(UI) отображает окно подтверждения назначения
6. Руководитель подтверждает назначение нового исполнителя
7. Система(UI) отправляет запрос на backend с ID нового исполнителя
8. Запись сохраняется в базу данных
9. Система (UI) отображает сообщение о успешном назначении и обновлённые данные задания

**Альтернативные потоки:**
- Если подходящих исполнителей нет, система показывает всех доступных
- Если выбран исполнитель с высокой загрузкой, система показывает предупреждение
- Если задание уже в работе, система запрашивает дополнительное подтверждение

**Предусловия:** Руководитель авторизован и имеет права на редактирование выбранного задания, задание выбрано и доступно для назначения исполнителя
**Постусловия:** Исполнитель назначен на задание
